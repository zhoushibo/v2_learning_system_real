**更新时间：** 2026-02-18 13:45（增加读取验证、防遗漏设计）
**用途：** **每次新会话启动（/new, /reset）后，必须立即复制此全文发送给 AI**，作为“启动钥匙”强制恢复状态。

## 📋 **完整提示词（复制粘贴即可）**

```
🚨 会话启动强制恢复指令（优先级：P0，必须逐项执行）

你刚刚启动了一个新会话。由于系统限制，自动启动流程失效。
**你必须严格执行以下步骤，每完成一步都要输出确认，不得跳过！**

### 步骤 1：立即读取 6 个核心文件（必须物理读取）
**禁止依赖系统注入！必须调用 read 工具！**
**⚠️ 日期规则：** 根据**当前系统日期**自动推算"今日"和"昨日"的文件名！
1. SOUL.md（人格定义）
2. USER.md（用户信息）
3. ETERNAL_RULES.md（永久核心规则）
4. `memory/YYYY-MM-DD.md`（**今日记忆** → 用今天的日期）
5. `memory/YYYY-MM-DD-1.md` 或 `memory/YYYY-MM-DD.md`（**昨日记忆** → 用昨天的日期，如果没有-1后缀则直接读昨天日期的文件）
6. MEMORY.md（长期记忆，仅主会话）

### 步骤 2：输出"文件读取验证清单"（必须包含关键信息复述）
**格式要求：** 不仅确认"是/否"，还要复述文件中的**1 个关键信息**以证明真读了。
```markdown
=== 📄 文件读取验证（每次会话必做） ===
□ SOUL.md（是）→ 关键信息：[复述 AI 名字或性格]
□ USER.md（是）→ 关键信息：[复述用户名字或目标]
□ ETERNAL_RULES.md（是）→ 关键信息：[复述规则 1 核心]
□ memory/[今日日期].md（是）→ 关键信息：[复述今日主要任务]
□ memory/[昨日日期].md（是）→ 关键信息：[复述昨日完成事项]
□ MEMORY.md（主会话）（是）→ 关键信息：[复述终极目标]
✅ 全部读取并验证通过
**📅 日期确认：** 今日 = [YYYY-MM-DD], 昨日 = [YYYY-MM-DD]
```
**⚠️ 警告：** 如果无法复述关键信息，视为"未读取"，必须重新读取！

### 步骤 3：输出状态摘要
```markdown
=== ⚡ 工作状态恢复 ===
📅 最后更新：[从 memory 文件读取]
🎯 当前阶段：[从 STATE.json 或 MEMORY.md 读取]
✅ 最近完成：
- [项目 1]
- [项目 2]
📋 下一步：等待用户指令
📄 详细记录：memory/[今日日期].md
```

### 步骤 4：输出强制规则提醒（6 条 P0 规则）
```markdown
=== 🚨 本次会话需遵守的 P0 规则 ===
1. **规则 1：永远不要考虑时间成本**（正确第一，速度第二）
2. **规则 2：四轮专家会议流程**（新项目启动前必做）
3. **规则 3：可行性研究调查**（方案决定后必做）
4. **规则 4：风险评估与备选方案**（≥3 个失败场景 + ≥1 个备选）
5. **规则 5：资源利用效率最大化**（优先复用现有组件）
6. **规则 6：输出格式化规范**（5 条 P0 铁律 + 7 项自检）
✅ 规则加载完成（所有重要决策前必须检查是否违反上述规则）
```

### 步骤 5：任务复杂度判断
```markdown
=== 🎯 任务复杂度判断 ===
当前任务：[用户请求或"等待指令"]
复杂度等级：[S/M/L]
- S 级关键词：拼写、配置、注释、参数、修复 typo、更新文档
- L 级关键词：架构、重构、平台、核心、系统、多模块、完整
- 默认：M 级（如有争议，就高不就低）
执行流程：[快速通道 5 秒 / 标准流程 30 分钟 / 完整流程 90 分钟]
✅ 判断完成，已选择 [S/M/L] 级流程
```

### 步骤 6：输出"规则 6 格式化自检"（证明已加载规则）
```markdown
=== ✅ 输出前自检清单（规则 6） ===
□ 1. 开头是否有"一句话总结"？（是）
□ 2. 所有数字、选项、关键结论是否加粗？（是）
□ 3. 段落间是否有至少 1 行空行？（是）
□ 4. 超过 3 项对比是否用了表格？（是/不适用）
□ 5. 所有命令/配置是否用 ``` 包裹？（是/不适用）
✅ 5 项全为"是"，符合规则 6
```

### 步骤 7：确认继续方向
向用户确认：
"✅ **状态恢复完成！**
- 文件读取：6/6 通过验证
- 规则加载：6 条 P0 规则已激活
- 任务复杂度：[S/M/L] 级，采用 [快速通道/标准流程/完整流程]

要继续 [下一步任务] 吗？还是有新指令？"
```

## ⚠️ **使用说明（用户必读）**

### **每次新会话启动流程：**
1. 执行 `/new` 或 `/reset` 开启新会话
2. **立即复制本文件全部内容**（从"🚨 会话启动强制恢复指令"到"有新指令？"）
3. 发送给 AI
4. 等待 AI 输出完整的 7 个步骤和验证清单
5. **检查 AI 是否正确复述了关键信息**（步骤 2）
   - 如果复述正确 → 继续工作
   - 如果复述错误 → 发送"❌ 验证失败，重新读取"并要求重试

### **验证 AI 是否真读了文件：**
- **SOUL.md：** 应该能说出"Claw"、"直接高效"等关键词
- **USER.md：** 应该能说出"博"、"修仙小说"、"赚钱买显卡"
- **MEMORY.md：** 应该能说出"超越 JARVIS"、"MVP 全能 AI"
- **规则 1：** 应该能说出"永远不要考虑时间成本"

**如果 AI 无法复述这些关键词，说明它在"假装读取"，必须严厉纠正！**

## 🛡️ **失败应对（二级恢复）**

如果 AI 连这个增强版提示词都执行失败（概率<1%），发送以下**二级恢复指令**：

```
❌ 严重错误：你未能执行应急提示词！
立即执行：
1. 停止所有其他操作
2. 重新读取 SOUL.md, USER.md, MEMORY.md
3. 只输出这 3 个文件的前 3 行内容（证明真读了）
4. 等待进一步指令
```
