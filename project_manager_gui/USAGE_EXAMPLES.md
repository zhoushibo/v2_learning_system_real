# OpenClaw项目管理器 - 快速使用示例

---

## 🚀 **快速启动**

### 1. 安装依赖
```bash
cd project_manager_gui
pip install PyQt5 psutil requests
```

### 2. 运行程序
```bash
python main.py
```

## 📋 **使用场景示例**

### 场景1：启动V2 MVP

**需求：** 启动OpenClaw V2异步架构的Gateway和Worker

**操作步骤：**
1. 运行程序 `python main.py`
2. 在项目列表中找到 "V2 MVP"
3. 点击 "启动" 按钮启动 Gateway
4. 点击 "启动" 按钮启动 Worker
5. 查看日志输出确认启动成功

**预期结果：**
```

```

## 🔧 **高级用法**

### 添加新项目

编辑 `config/projects.json` 添加新项目：

```json
{
  "projects": [
    {
      "id": "my_project",
      "name": "我的项目",
      "type": "web",
      "path": "C:\\path\\to\\my_project",
      "description": "我的Web应用",
      "components": [
        {
          "id": "server",
          "name": "Web服务器",
          "command": "python server.py",
          "working_dir": "C:\\path\\to\\my_project",
          "port": 5000,
          "health_check": "http://127.0.0.1:5000/health"
        }
      ]
    }
  ]
}
```

重启程序后即可看到新项目。

## ⚠️ **常见问题**

### Q1: 启动失败怎么办？

**A:** 查看日志输出，常见原因：
- 端口被占用
- 工作目录不存在
- 启动命令错误

### Q2: 停止失败怎么办？

**A:** 尝试：
1. 等待几秒钟让进程优雅退出
2. 如果仍然停止失败，手动在任务管理器中结束进程

### Q3: 托盘图标不显示？

**A:** 检查Windows设置：
1. 右键任务栏 → 任务栏设置
2. 选择"打开或关闭系统图标"
3. 确保"通知区域"已开启

## 📊 **性能提示**

### 优化建议
1. **只启动需要的组件** - 不要启动不使用的组件
2. **及时停止** - 用完及时停止进程释放资源
3. **定期清理** - 关闭不需要的程序

### 资源占用
- **内存：** ~50MB（空闲）
- **CPU：** <1%（空闲）
- **网络：** 无（仅本机通信）

## 🎯 **最佳实践**

1. **启动顺序**
   - 先启动Gateway
   - 再启动Worker
   - 依赖关系按顺序启动

2. **停止顺序**
   - 先停止Worker
   - 再停止Gateway
   - 避免资源泄漏

3. **监控日志**
   - 定期查看日志输出
   - 关注错误信息
   - 及时处理异常
