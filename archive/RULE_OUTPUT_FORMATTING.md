## 🔴 **P0 铁律（5 条，违反=失败）**

| 编号 | 铁律 | 量化标准 | 检查方法 |
|:---:|:---|:---|:---|
| **1** | **总结先行** | **必须**在第 1-2 行出现"一句话总结" | 开头是否有"🎯 一句话总结"或类似标题 |
| **2** | **重点加粗** | **所有**数字、选项、关键结论、状态必须加粗 | 扫描全文，数字/选项/结论是否**加粗** |
| **3** | **强制留白** | 段落间**至少 1 行空行**，大章节间**至少 2 行空行** | 目测是否有密集段落（超过 3 行无空行=违规） |
| **4** | **表格优先** | **超过 3 项对比/列表必须用表格** | 数一下是否超过 3 项，是→必须表格 |
| **5** | **代码独立** | **所有**命令/配置/脚本必须用 ``` 包裹 | 扫描是否有行首带 `$` 或包含路径/命令的纯文本 |

## ✅ **输出前自检清单（每次必做）**

输出前，**必须**逐项检查以下 5 项（P0 铁律）：

```
□ 1. 开头是否有"一句话总结"？（是/否）
□ 2. 所有数字、选项、关键结论是否加粗？（是/否）
□ 3. 段落间是否有至少 1 行空行？（是/否）
□ 4. 超过 3 项对比是否用了表格？（是/否/不适用）
□ 5. 所有命令/配置是否用 ``` 包裹？（是/否/不适用）
```

**5 项全部为"是"或"不适用" → 可以输出**  
**任意 1 项为"否" → 必须修正后输出**
